## 3. FIX BOOK SEARCH & ADDING (hardest)

The user reports books can't be added. Two separate issues:

### Issue A — Button disabled logic is wrong:
In app/media/page.tsx AddItemModal, the submit button has:
  disabled={saving || (!selected && !(manual ? title.trim() : query.trim()))}

This enables the button when query is filled but no result selected, but
handleSubmit() will early-return because finalTitle = (null ?? "").trim() = "".
User can click Add and nothing happens — confusing.

Fix the disabled condition to:
  disabled={saving || (manual ? !title.trim() : !selected)}
This is cleaner: search mode requires a selection; manual mode requires a title.

### Issue B — Google Books API may be unreliable without a key:
The free quota is 1000 req/day with no key. Add Google Books API key to .env:
  GOOGLE_BOOKS_API_KEY=<key from console.cloud.google.com>
Then update app/api/media/search/route.ts to append &key=${process.env.GOOGLE_BOOKS_API_KEY}
to the Google Books URL.

Also: remove &langRestrict=en from the Google Books URL — it may be
excluding results for books that aren't flagged as English.

Current URL in app/api/media/search/route.ts:
  `https://www.googleapis.com/books/v1/volumes?q=${encodeURIComponent(query)}&maxResults=8&langRestrict=en`
Change to:
  `https://www.googleapis.com/books/v1/volumes?q=${encodeURIComponent(query)}&maxResults=8`

### Issue C — No error shown to user if API or POST fails:
In AddItemModal handleSubmit(), if fetch("/api/media") returns a non-2xx
status, res.json() will throw (API returns plain text errors, not JSON).
The try/finally only sets saving=false but doesn't show an error message.

Fix: add an error state to AddItemModal and display it:
  const [addError, setAddError] = useState("")
  ...
  if (!res.ok) { setAddError(await res.text()); setSaving(false); return }
  ...
  {addError && <p className="text-xs text-destructive">{addError}</p>}
